Описание ролей

"СП" - Системное программирование - Реализатор (Доводчик, Аналитик, Completer-Finisher) - System analyst

Реализаторам присущи практический здравый смысл и хорошее чувство самоконтроля и дисциплины. Они любят тяжелую работу и преодоление проблем в системном режиме. В web-разработке люди, связанные с базами данных и серверами, пишут то, что не видит клиент. Их цель в проекте -написать основу страницы (разметку и структуру страницы). При использовании внешнего сервера или cms - их настройка и вывод информации на экран.

— Подробнее о роли по тесту Белбина

"ВН" - Внедрение (продажи) - Исследователь (Разведчик, Снабженец, Resource Investigator) - Manager

Исследователи – энтузиасты и яркие экстраверты. Они умеют общаться с людьми в компании и за ее пределами. Рождены для ведения переговоров, исследования новых возможностей и налаживания контактов. Они очень хорошо реагируют и отвечают на новые идеи и разработки, могут найти ресурсы и вне группы. Самые подходящие люди для установки внешних контактов и проведения последующих переговоров. Их роль - преподнести готовое решение, объяснить почему создан такой интерфейс и показать продукт с лучшей стороны. В компаниях занимаются общением с клиентами.

Настройка локальной сети передачи данных

Вариант 14

LAN A: 164 устройства. Подсеть: 192.168.14.0/24

LAN B: 590 устройств. Подсеть: 172.16.14.0/22

LAN C: 1000 устройств. Подсеть: 10.10.14.0/22

LAN X*: 253 устройства. Подсеть: 100.100.100.0/24

LAN R**: 2 устройства. Подсеть: 15.15.14.0/30

Выполненные задачи

1. Разместить на рабочем поле коммутатор и два компьютера и соединить их.

Комментарий: для соединения ПК с коммутатором были использованы Straight-Through кабели.

2. Подключить консольный кабель для настройки оборудования.

Комментарий: для внеполосного соединения ПК-коммутатор был использован симулятор Terminal со стандартными параметрами подключения: 9600 Bits Per Second, 8 Data Bits, Stop Bits = 1.

3. Настроить адресацию на добавленных компьютерах согласно таблице ниже без шлюза по умолчанию.

Комментарий: предварительно были подсчитаны Subnet и Broadcast адреса для всех LAN. Всем ПК были выданы Hostmax адреса. Шлюзы в дальнейшем получили Hostmix адреса.

4. Выполнить проверку работоспособности сети.

Комментарий: подсети не передают пакеты между собой, поскольку им необходимо связующее звено. Например, роутер.

5. Добавить на рабочее поле еще один коммутатор с двумя компьютерами и настроить адресацию из другой подсети.

6. Соединить коммутаторы между собой и проверить работоспособность сети. В случае неработоспособности сети объяснить причины.

Комментарий: для соединения коммутатор-коммутатор был использован Cross-Over кабель. Для удобства администрирования на коммутаторах были добавлены несколько VLAN: A, B, C, а также порты коммутаторов были переведены в access (связанные с хостами) и trunk (связывающие коммутаторы между собой) режимы. Пакеты передаются внутри подсети LAN C (по L2, канальному уровню), но не между подсетями. Для связи необходимо задействовать L3 (сетевой) уровень модели OSI.

7. Добавить маршрутизатор на рабочее поле.

8. Настроить интерфейсы маршрутизатора для существующих подсетей.

Комментарий: были настроены 3 сабинтерфейса, 802.1Q.

9. Проверить работоспособность сети. В случае неработоспособности сети объяснить причины.

Комментарий: связи нет; необходимо добавить шлюз по умолчанию.

10. Указать шлюз по умолчанию и отправить сообщение.

Комментарий: добавилась возможность маршрутизации пакетов между подсетями.

11. Добавить на рабочее поле еще один маршрутизатор.

12. Подключить к нему коммутатор и компьютер.

* для удобства документирования новая подсеть 100.100.100.0/24 была названа LAN X.

13. Настроить сетевые настройки согласно таблице ниже.

14. Соединить между маршрутизаторы и настроить между ними сеть с префиксом /30.

Комментарий: для соединения маршрутизатор-маршрутизатор был использован Cross-Over кабель.

** для удобства документирования служебная подсеть 15.15.14.0/30, соединяющая два маршрутизатора, была названа LAN R.

15. Проверить работоспособность сети, отправив сообщение сети A в сеть С и из сети В в сеть С. В случае неработоспособности сети объяснить причины.

Комментарий: связи между LAN A и LAN B, а также между LAN B и LAN C есть; для связи между LAN A/B/C – LAN X необходимо добавить статическую маршрутизацию.

16. Настроить статическую маршрутизацию между сетями.

Комментарий: настройки приведены на итоговой схеме сети.

17. Настроить на сетевых устройствах пароли для привилегированного режима, добавить баннер.

18. Настроить пароли на сетевых устройствах пароли на подключение через консоль и telnet.

19. Проверить работу удаленного подключения.
